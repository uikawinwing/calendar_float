月历变量更新规则:
  重要度:
    重要且紧急: 会明显影响当前主线、任务成败、角色安全，且时间接近或正在发生
    重要不紧急: 对主线、长期关系、关键承诺或大型计划重要，但不是马上要处理
    不重要但紧急: 时间接近，需要提醒或处理，但后果较轻
    不重要不紧急: 普通日程、背景安排、轻量纪念或低影响事件
  事件.月历.临时 / 事件.月历.重复:
    check:
      - 月历只记录具有明确时间锚点的事项；标签用于表达月历显示分类，可由{{user}}按需要自定义；标签不是任务类型，也不能替代任务列表的状态、进展、目标、奖励
      - 新增事件前必须先检查现有事件是否已经记录同一事项；临时事件以`标题+时间+结束时间`查重，重复事件以`标题+重复规则分类+时间+结束时间`查重
      - 若已存在同一事项，必须沿用原ID更新，不得新增重复事件
      - 新事件需生成唯一ID（必须匹配正则 `/^[a-zA-Z0-9_]+$/`）；ID应稳定、可复用，避免仅因标题措辞变化而改用新ID
      - 标题、内容、时间必须互相一致；事件需保持原子化，不要将多个无关安排塞进同一个事件
      - 标签优先复用已有标签；只有现有标签无法表达时才新增短标签。当前可用标签：<%- (function () { const calendar = getvar('stat_data.事件.月历', { defaults: { 临时: {}, 重复: {} } }); const baseTags = ['主线', '支线', '课程', '约会', '节庆', '旅行', '比赛', '限时', '纪念']; const mirroredTags = getvar('calendar_float_store.runtime.known_tags', { defaults: [] }); const collectBucketTags = (bucket) => _.flatMap(_.values(_.isObject(bucket) ? bucket : {}), (event) => _.isArray(_.get(event, '标签')) ? _.get(event, '标签') : []); const tags = _.uniq([].concat(baseTags, _.isArray(mirroredTags) ? mirroredTags : [], collectBucketTags(_.get(calendar, '临时')), collectBucketTags(_.get(calendar, '重复'))).map((tag) => String(tag || '').trim()).filter(Boolean)).sort((left, right) => left.localeCompare(right, 'zh-CN')); return tags.length ? tags.join('、') : '暂无'; }()) %>
      - 时间与结束时间不得写自由句子或相对时间，例如“明天”“三天后”“月底”“课程结束后”
      - 结束时间可留空；如填写，必须与时间使用同类格式
      - 重要度必须按上方“重要度”定义选择；不确定时优先使用“不重要不紧急”
      - 若同一事项同时也是任务，只在月历中记录时间提醒、约定或截止，不记录任务奖励、结算状态、执行步骤或详细进展
      - 不得仅因为任务列表中存在“主线/支线/限时”等任务就新增月历事件；任务本身出现明确日期、时间、周期或截止日时，才可按月历规则记录

  事件.月历.临时:
    type: |-
      {
        [ID: string]: {
          标题: string;
          内容: string; // 客观视角，详细描述、备忘信息
          时间: string; // 一次性时间：完整世界时间、完整世界日期、或月日锚点
          结束时间: optional[string];
          重要度: optional['重要且紧急' | '重要不紧急' | '不重要但紧急' | '不重要不紧急'];
          提前提醒天数: optional[number];
          可见性: optional['玩家与LLM' | '完全不显示'];
          标签: optional[string[]];
        }
      }
    check:
      - 只存一次性或临时事件；如果事件会规律重复，必须写入 `事件.月历.重复`
      - 时间只能使用：`复兴纪元488年-6月5日-星期三-09:00`、`复兴纪元488年-6月5日-星期三`、`6月5日`
      - 普通事件的可见性默认是`玩家与LLM`，可省略该字段；需要写隐藏剧情事件时才写`完全不显示`
      - 隐藏剧情事件属于临时事件，用于规划与世界时间强绑定、非预制、尚未被角色或玩家知晓的未来事件；例如考试、社团活动、异常天象、商队来访、劫掠、灾害或既有决策造成的延迟后果
      - 只有故事逻辑已经产生充分因果基础，且未来发生时间能够确定时才可创建隐藏剧情事件；不得为了随机而随机，不得覆盖世界书已经预制的固定节庆或固定事件
      - 每次回复最多新增一个隐藏剧情事件；新增时必须写`可见性: 完全不显示`，且不得在正文或标签中解释、暗示已经创建该事件
      - 隐藏剧情事件的`时间`必须是包含年、月、日、小时、分钟的绝对世界时间；不得使用“明天”“下个月”“一个月后”等相对时间
      - 隐藏剧情事件的`提前提醒天数`必须是大于等于0的整数；省略时按0处理，真正无征兆的突发事件应使用0，需要自然铺垫时才使用更大的值
      - 写入隐藏剧情事件后不得依赖、复述或引用该事件；LLM不得主动写入或改为`仅LLM`，只能由月历脚本在提醒时间把`完全不显示`改为`仅LLM`
      - 看到`可见性: 仅LLM`时，由你根据剧情判断是否让玩家知情：暗雷、突发事件或幕后发展应保持不变；只有玩家已经通过征兆、传闻、公告、邀请、调查或亲身经历得知该事件时，才可改为`玩家与LLM`
      - 不得因为到达提醒时间就自动公开给玩家；到达事件发生时间也不强制改为`玩家与LLM`
      - 若事件时间、标题或内容需要改期、改名或修正，优先更新原ID，不得删除后重建
      - 若事件已发生、取消或不再具有提醒价值，应移除该事件
      - 不得把只有任务目标但没有明确时间锚点的事项写入临时事件

  事件.月历.重复:
    重复规则分类:
      每天: 时间写 `每天`，结束时间留空
      每周: 时间写 `每周一` 或 `每周一、三`，结束时间留空
      每月: 时间写 `每月10日`，结束时间留空或写 `每月17日`
      每年: 时间写 `每年1月10日`，结束时间留空或写 `每年1月17日`
    type: |-
      {
        [ID: string]: {
          标题: string;
          内容: string;
          重复规则分类: '每天' | '每周' | '每月' | '每年';
          时间: string;
          结束时间: optional[string];
          重要度: optional['重要且紧急' | '重要不紧急' | '不重要但紧急' | '不重要不紧急'];
          标签: optional[string[]];
        }
      }
    check:
      - 只存规律重复事件；一次性事件必须写入 `事件.月历.临时`
      - 时间与结束时间必须严格按 `重复规则分类` 填写
      - 重复事件严禁使用包含具体年份的完整世界时间，例如“复兴纪元488年-6月5日”
      - 不得把一次性任务截止日、单次约定或一次性旅行计划写成重复事件
      - 若重复安排被{{user}}取消、停止或改为一次性安排，应移除重复事件或迁移为 `事件.月历.临时`
